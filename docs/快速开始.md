# 快速开始

## 🎯 5分钟上手

### 第一步：启动项目

```bash
# 安装依赖
npm install

# 启动开发服务器
npm run dev
```

访问：http://localhost:5173

### 第二步：选择模板

在顶部下拉框中选择一个页面模板：

**标准列表页**（最常用）
- 结构：搜索区 + 表格 + 分页
- 组件：h-page-search + h-page-table
- 适合：数据查询、列表管理等场景

**左树右表**
- 结构：左侧树形导航 + 右侧表格列表
- 适合：分类数据、组织架构等场景

**多Tab列表**
- 结构：顶部Tab切换 + 列表内容
- 适合：按状态分组的数据

> 💡 **模板提示**：这些组件已经内置了响应式布局、表头固定、高低频搜索等功能，无需手动配置！

### 第三步：添加组件

#### 添加搜索组件

1. 从左侧"搜索组件"分类找到 `el-input`
2. 拖拽到中间预览区的**搜索区 slot**
3. 松开鼠标，组件自动添加

#### 添加更多组件

按同样方式添加：
- `el-select`（下拉选择）
- `el-date-picker`（日期选择）
- 更多...

### 第四步：配置组件

点击预览区的组件，右侧会显示配置面板。

#### 基础配置
```
字段名：userName
显示标签：用户名
占位符：请输入用户名
可清空：✓
```

#### 接口绑定

**提交参数配置**：
1. 点击"添加接口"
2. 选择用途：`提交主接口`
3. 参数名：`userName`
4. 默认值：``（空）

**下拉选项配置**（针对 el-select）：
1. 点击"添加接口"
2. 选择用途：`获取下拉选项`
3. URL：`/api/status-options`
4. Method：`GET`
5. 触发时机：`页面加载`
6. 数据路径：`data.items`
7. Label字段：`statusName`
8. Value字段：`statusCode`

### 第五步：配置表格

1. 点击预览区的表格区域
2. 右侧切换到"接口管理" Tab
3. 配置主数据接口：
   ```
   URL: /api/users
   Method: POST
   数据路径: data.rows
   总数路径: data.total
   ```
4. 添加表格列（手动或AI推断）

### 第六步：生成代码

1. 点击顶部"生成代码"按钮
2. 等待 5-10 秒
3. 查看生成结果
4. 点击"复制代码"或"下载文件"

## 🔧 进阶配置

### 级联下拉框

**场景**：选择团队后，组织下拉框自动加载对应的组织列表。

#### 配置"团队"下拉框

1. 添加 `el-select` 组件
2. 基础配置：
   ```
   字段名：teamId
   显示标签：所属团队
   ```
3. 添加接口1（获取选项）：
   ```
   用途：获取下拉选项
   URL: /api/teams
   触发时机：页面加载
   数据路径: data.list
   ```
4. 添加接口2（提交参数）：
   ```
   用途：提交主接口
   参数名：teamId
   ```
5. AI提示词：
   ```
   当团队改变时，需要清空组织字段并重新加载组织选项
   ```

#### 配置"组织"下拉框

1. 添加 `el-select` 组件
2. 基础配置：
   ```
   字段名：orgId
   显示标签：所属组织
   ```
3. 添加接口1（获取选项）：
   ```
   用途：获取下拉选项
   URL: /api/organizations
   触发时机：watch:teamId  ← 关键！
   依赖参数：teamId
   数据路径: data.items
   ```
4. 添加接口2（提交参数）：
   ```
   用途：提交主接口
   参数名：orgId
   ```
5. AI提示词：
   ```
   依赖团队字段，只有选择团队后才加载组织选项
   ```

### 日期范围转换

**场景**：前端使用日期范围选择器，后端需要 startTime 和 endTime 两个字段。

1. 添加 `el-date-picker` 组件
2. 基础配置：
   ```
   字段名：dateRange
   显示标签：创建时间
   类型：daterange
   ```
3. 添加接口（提交参数）：
   ```
   用途：提交主接口
   参数名：不填（因为需要转换）
   数据转换：日期范围拆分
     - 开始字段：startTime
     - 结束字段：endTime
     - 格式：YYYY-MM-DD
   ```
4. AI提示词：
   ```
   日期范围需要拆分成 startTime 和 endTime 两个参数提交给后端
   ```

### 全局提示词

在右侧"页面设置" Tab 的最底部，可以编写全局AI提示词：

```
这是一个用户管理列表页。

业务规则：
- 团队和组织是级联关系，选择团队后才能选择组织
- 状态字段有"启用"和"停用"两个值
- 日期范围需要转换成 startTime 和 endTime 格式
- 分页参数使用 pageNo 和 pageSize

特殊逻辑：
- 导出功能使用当前搜索条件
- 重置时需要清空所有字段并重新加载表格
```

## 💡 最佳实践

### DO（推荐）

✅ **字段命名使用驼峰式**
```
userName、createTime、statusCode
```

✅ **标签文字简洁明了**
```
用户名、创建时间、状态
```

✅ **占位符给出明确提示**
```
请输入用户名、请选择状态
```

✅ **接口绑定要完整**
- 下拉框：一定要绑定选项接口
- 所有搜索组件：都要绑定提交参数

✅ **级联关系要明确**
- 使用 `watch:` 触发时机
- 在AI提示词中说明依赖关系

✅ **AI提示词要具体**
```
✓ "团队改变时清空组织并重新加载"
✗ "注意依赖关系"
```

### DON'T（避免）

❌ **字段名使用缩写**
```
usrNm、ct、sts
```

❌ **标签文字冗余**
```
请输入用户名称  ← 占位符中说
```

❌ **忘记绑定接口**
- 下拉框没有选项接口 → 运行时报错
- 搜索组件没有提交参数 → 搜索无效

❌ **级联关系配置错误**
- 触发时机选错 → 不会自动加载
- 忘记依赖参数 → 接口调用失败

❌ **AI提示词太模糊**
```
"这是一个列表页"  ← 没有任何帮助
```

## 🐛 常见问题

### Q: 预览区没有显示组件？

**A**: 检查：
1. 是否拖到了正确的 slot 区域（会高亮显示）
2. 查看浏览器控制台是否有错误

### Q: 生成的代码有错误？

**A**: 常见原因：
1. 接口绑定不完整（缺少选项接口或提交参数）
2. 级联关系配置错误（触发时机或依赖参数）
3. AI提示词太模糊

**解决**：
- 仔细检查配置
- 补充更详细的AI提示词
- 或手动修改生成的代码

### Q: AI 生成失败？

**A**: 不用担心：
- 工具会自动降级到模板渲染
- 生成的代码同样可用
- 可能需要手动完善部分逻辑

### Q: 如何保存配置？

**A**: 
1. 点击顶部"保存配置"按钮
2. 导出 JSON 文件到本地
3. 下次使用时点击"导入配置"
